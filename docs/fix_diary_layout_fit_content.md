# 日記レイアウト調整計画 (Refining Layout)

## 現状の問題

ユーザー様からの「直っていない」というご指摘について。
現在は `max-width: 800px` で幅を確保しているため、文字数が少ない（一行が短い）場合、**「見えない800pxの箱」の左端** に文字が表示されてしまいます。
その結果、画面全体で見ると「左に寄っている」ように見えてしまいます。

## 解決策

箱の幅を「固定（800px）」ではなく、「**文字の長さに合わせる (`fit-content`)**」ように変更します。
これにより、文字が短いときは箱も小さくなり、その小さい箱が画面のド真ん中に配置されるため、見た目が綺麗に中央に寄ります。

### 変更内容 (`style.css`)

```css
.notebook-content {
  /* 変更前 */
  /* width: 90%; */
  /* max-width: 800px; */

  /* 変更後 */
  width: fit-content;  /* 中身（文字）に合わせて箱のサイズを決める */
  max-width: 90%;      /* ただし画面からはみ出さないように制限 */
  margin: 2rem auto;   /* 上下余白＋左右自動（これで中央に来る） */
  
  /* レイアウト崩れ防止 */
  min-width: 300px;    /* 極端に小さくなりすぎないように最低幅を確保 */
  padding: 1.5rem;     /* 文字の周りに少し余白を持たせる */
}
```

## 手順

1. `style.css` を上記の内容に書き換える。
2. `base.html` のCSSバージョンを `v=14` に上げて反映させる。
