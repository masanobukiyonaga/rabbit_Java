# 日記本文のレイアウト変更計画 (Implementation Plan)

## ゴール

日記の本文 (`.notebook-content`) を、画面（または親要素）の中央に配置しつつ、文章の左端（頭）は揃った状態にする。

## 現状の課題

現在は `width: 100%` または幅指定なしで、親要素いっぱいに広がっているため、左端から始まっているように見える（あるいは中央寄せ指定がない）。

## 変更内容

`src/main/resources/static/css/style.css` を編集します。

### `.notebook-content` への変更

1. **幅の制限**: `max-width` を設定する（例: 80% や 600px）。これがないと「中央」の概念が生まれないため。
2. **中央配置**: `margin: 0 auto;` を追加して、ブロック自体を中央に寄せる。
3. **ブロック化**: `display: block;` (デフォルト) を維持。

```css
.notebook-content {
  /* 既存のスタイル */
  font-family: ...;
  
  /* 追加するスタイル */
  width: 90%;       /* スマホも考慮して％指定 */
  max-width: 800px; /* PCで見やすくなる最大幅 */
  margin: 2rem auto; /* 上下に余白、左右は自動（中央寄せ） */
}
```

## 検証方法

1. サーバーを再起動する。
2. 日記詳細ページを開き、本文が画面中央にまとまっているか確認する。
3. 文章の「頭（左端）」が揃っていることを確認する（`text-align: center` になっていないこと）。
