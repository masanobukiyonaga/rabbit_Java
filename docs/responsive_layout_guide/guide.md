# 自分以外の画面サイズにも対応する（レスポンシブデザイン）方法

「自分の画面では綺麗だけど、他の人の画面（スマホや大きなモニター）だとズレる」という問題を解決するための、**「相対値（%やrem）」** を使ったレイアウトの決め方を解説します。

## 基本的な考え方

これまでの「350pxずらす」といった**固定値（絶対値）**の指定から、
「画面の幅の半分」「残りスペースの真ん中」といった**相対値**の指定に切り替えます。

---

## 使えるテクニック集

### 1. 画面の真ん中に置くなら `Flexbox` (最強)

「何pxずらす」と計算しなくても、自動で「ど真ん中」に置いてくれます。

```css
.container {
  display: flex;           /* Flexboxを使う宣言 */
  justify-content: center; /* 左右の真ん中に寄せる */
  align-items: center;     /* 上下の真ん中に寄せる */
  min-height: 100vh;       /* 高さを画面いっぱい(100%)にする */
}
```

**メリット:** 画面サイズが変わっても常に真ん中をキープします。
今回の「日記リスト」や「ログイン画面」はこれで実装されています。

### 2. 幅は `px` ではなく `%` と `max-width`

「幅1000px」と決め打ちすると、スマホ（幅400pxくらい）でははみ出します。
かといって「幅90%」だけにすると、超巨大モニターで横長になりすぎます。

**正解のパターン:**

```css
.card {
  width: 90%;          /* 基本は画面の90%を使う（スマホ対応） */
  max-width: 1000px;   /* でも最大1000px以上は広がらない（PC対応） */
}
```

これで、スマホでもPCでも「いい感じ」の幅になります。

### 3. 余白は `rem` (文字サイズ基準)

`px` で余白を作ると、画面サイズによって狭く見えたり広く見えたりします。
`rem` という単位を使うと、「文字の大きさ」を基準にするのでバランスが崩れにくいです。

* `1rem` = だいたい16px（標準的な文字サイズ1つ分）
* `padding: 3rem;` = 文字3つ分の余白

---

## 今回の「右にずれる問題」の正体

以前の設定では、以下のような矛盾が起きていました。

1. **Flexbox:** 「画面の真ん中に置いて！」
2. **Padding:** 「そこからさらに右に350pxずらして！」
3. **結果:** 真ん中より右に350pxずれて表示される

**解決策（今回やったこと）:**
「右にずらす」をやめて、「親要素（mainタグ）の中で真ん中に置く」だけにしました。
（※実はすでに `body:not(.home) main` という場所で、サイドバーを避けるための余白設定（margin-left）が自動で入っています。なので、素直に「真ん中」と指定するだけで、サイドバーと被らない正しい位置に来るようになっています）

## 今後のレイアウトのコツ

1. **位置決め:** `margin-top: 100px` のような固定値で位置調整するのは最終手段にします。まずは `display: flex` か `margin: 0 auto`（左右自動）で大まかな位置を決めます。
2. **サイズ:** `width` は `%` か `auto` にし、広がりすぎ防止に `max-width` を使います。
3. **サイドバー:** サイドバーがあるレイアウトの場合、「サイドバーの幅分ずらす」のではなく、「コンテンツエリア（main）の余白」として設定し、中身はその中で自由にレイアウトするのが鉄則です（今の `style.css` は既にそうなっています！）。
